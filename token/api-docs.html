🔐 认证方式

所有API请求都需要在请求头中包含您的用户Token：

Authorization: Bearer YOUR_USER_TOKEN

📋 接口列表

GET 获取OAuth授权链接

/api/external/auth-url

请求参数

无需额外参数，直接请求即可获取标准授权链接。

响应示例

{ "authorize_url": "https://auth.augmentcode.com/authorize?response_type=code&code_challenge=xxx&client_id=v&state=xxx&prompt=login" }
📝 使用示例：
curl -X GET "http://localhost:3000/api/external/auth-url" \ -H "Authorization: Bearer YOUR_USER_TOKEN"

POST 完成OAuth授权

/api/external/complete-auth

请求体参数

参数名 类型 必需 说明
code string 是 OAuth授权码
state string 否 状态参数验证
tenant_url string 是 租户URL（必填）

响应示例

{ "success": true, "token": "access_token_value", "tenant_url": "https://tenant.augmentcode.com", "token_info": { "id": 123, "created_at": "2025-01-01T00:00:00Z" }, "user": { "uuid": "user_uuid", "email": "user@example.com" } }
📝 使用示例：
curl -X POST "http://localhost:3000/api/external/complete-auth" \ -H "Authorization: Bearer YOUR_USER_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "code": "authorization_code_from_callback", "state": "state_value", "tenant_url": "https://your-tenant.augmentcode.com" }'

❌ 错误响应

当请求失败时，API会返回以下格式的错误响应：

{ "success": false, "error": "错误描述", "details": "详细错误信息" }

🔧 常见错误码

状态码 说明
401 未提供Token或Token无效
400 请求参数错误
500 服务器内部错误
